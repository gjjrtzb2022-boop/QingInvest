# sync-reports 目录说明

## 当前策略

- 根目录只保留 `latest-*.json`（看最新状态）
- 历史报告归档到 `archive/日期/`，并按报告类型再分类

## 归档分类（示例）

- `archive/2026-03-03/check-env/`
- `archive/2026-03-03/check-migrations/`
- `archive/2026-03-03/verify-rls/`
- `archive/2026-03-03/verify-content-sync/`
- `archive/2026-03-03/sync-content/`
