## 六、网站工程与后端基建补充（2026-03-02 晚间）

> 本节补充的是同一天（2026-03-02）晚间的工程化改造，重点面向“可上线、可验收、可排障”。

### 1) 发布门禁升级（CI）

- 工作流改造：`.github/workflows/deploy-pages.yml`
- 新增并串联门禁顺序：
  1. `check:env --target=dev`
  2. `check:migrations`
  3. `verify:rls`
  4. `validate:articles`
  5. `verify:content-sync --ci`
  6. `sync:content --dry-run --ci`
  7. `typecheck`
  8. `lint`
  9. `build:articles`
  10. `build`
- CI 中增加 `postgres:16` 服务，支持迁移与权限校验在流水线内自动执行。

### 2) 迁移与回滚演练工具补齐

- 新增脚本：`tools/check-migrations.mjs`
- 核心能力：
  - 在临时数据库中全量执行 migration；
  - 进行“N-1 重建”回滚演练并做 schema 指纹对比；
  - 输出可审计报告到 `raw/sync-reports/`。
- 相关命令：`npm run check:migrations`

### 3) 环境检查与 RLS 自动验收

- 新增脚本：`tools/check-env.mjs`
  - 按 `dev/prod` 检查关键变量是否齐全，并输出报告。
- 新增脚本：`tools/verify-rls.mjs`
  - 在临时库内构造 A/B 用户并验证“公有可读、私有隔离”策略；
  - 当前校验项为 7 项，支持 CI 阻断。
- 相关命令：
  - `npm run check:env -- --target=dev|prod`
  - `npm run verify:rls`

### 4) 同步链路性能优化（状态/批注）

- 阅读状态：从“单次点击即单条写库”升级为“本地即时 + 延迟批量上送”。
  - 文件：`lib/client/article-user-state.ts`
- 批注：从即时直写改为“本地先落盘 + 队列批量同步（新增/删除）”。
  - 文件：`lib/client/article-annotations-store.ts`
- 云端桥接层统一：`lib/client/content-sync-bridge.ts`
- 结果：交互响应更稳定，弱网下页面可继续操作。

### 5) 可观测能力首版落地

- 新增：`lib/client/sync-observability.ts`
  - 记录状态/批注同步的成功率、失败数、平均耗时（24h窗口）。

  - 文件：`components/settings/settings-center.tsx`
  - 展示字段：总次数、失败次数、平均耗时、最近失败时间。
