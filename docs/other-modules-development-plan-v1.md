# 其他模块开发方案 V1（非文章库/非用户中心）

## 1. 目标与边界

### 1.1 目标
- 在现有文章库稳定基础上，落地 5 个业务模块的可用 MVP：`选股器`、`股票分析`、`财务仪表盘`、`价格预警`、`AI问答助手`。
- 先保证本地开发可跑、功能闭环可验收，再考虑线上部署与规模化。
- 模块之间使用统一数据层，避免后续重复开发。

### 1.2 本阶段包含
- 各模块页面从占位版升级为可交互版本。
- 增加统一的行情/财务数据访问层（Provider 适配）。
- 增加最小可用的策略保存、预警规则、问答引用能力。

### 1.3 本阶段暂不做
- 正式商业化计费、权限套餐。
- 高频实时行情（逐笔/秒级）与量化回测引擎。
- 复杂多代理 AI 工作流（先做单模型 + 检索增强）。

## 2. 当前现状（2026-03-03）

- `app/screener/page.tsx`：选股器占位页
- `app/analysis/page.tsx`：股票分析占位页
- `app/dashboard/page.tsx`：财务仪表盘占位页

结论：业务模块 UI 已有入口，但功能基本未实现，适合按阶段并行推进。

## 3. 总体技术方案

### 3.1 架构分层
- `数据接入层`：统一封装行情/财务数据 Provider（先单 Provider，后可扩展多 Provider）。
- `指标计算层`：估值分位、成长质量、风险评分、预警触发判断。
- `业务服务层`：选股、分析、预警、问答检索服务。
- `展示层`：Next.js 页面 + 组件。

### 3.2 数据源策略（本地优先）
- 阶段内先使用“稳定可调用的单一数据源 + 本地缓存”。
- 加 `Provider Adapter` 接口，后续可替换为正式商业数据源。
- 对外部数据失败做降级：展示最近快照，不阻断页面。

### 3.3 存储策略（新增）
- 用户策略与规则：Supabase（与现有用户体系一致）。
- 行情/财务快照：先本地缓存（JSON/SQLite 均可），后续迁移到 Supabase 表或独立时序库。
- AI 知识源：文章库为主，后续可接研究笔记。

## 4. 模块拆解（MVP）

## 4.1 选股器（优先级 P0）
- 核心功能：
  - 多条件筛选：行业、估值（PE/PB 分位）、盈利能力（ROE/毛利率）、成长性（营收/利润增速）、风险项（负债率等）。
  - 结果排序：估值优先/质量优先/成长优先。
  - 候选池保存：保存个人筛选方案。
- 验收标准：
  - 用户可完成“设条件 -> 得结果 -> 保存方案 -> 复用方案”全流程。

## 4.2 股票分析（优先级 P0）
- 核心功能：
  - 单票分析卡：估值区间、历史分位、关键财务指标、风险提示。
  - 双票/三票对比：核心指标同屏对比。
- 验收标准：
  - 输入股票代码后 3 秒内返回结构化分析（允许基于缓存数据）。

## 4.3 财务仪表盘（优先级 P1）
- 核心功能：
  - 财务趋势图：营收、利润、现金流、ROE。
  - 关键健康指标：偿债、运营效率、盈利质量。
- 验收标准：
  - 至少支持 1 只股票完整展示近 3~5 年核心财务趋势。

## 4.4 价格预警（优先级 P1）
- 核心功能：
  - 规则创建：价格阈值、估值阈值、股息率阈值。
  - 触发记录：预警历史与状态。
- 验收标准：
  - 可新增/停用/删除规则，触发后在页面可见。

## 4.5 AI问答助手（优先级 P1）
- 核心功能：
  - 基于文章库检索问答（RAG）。
  - 回答必须附引用来源（文章标题 + 链接）。
- 验收标准：
  - 至少 5 类问题可稳定回答，且每条回答含来源。

## 5. 分阶段实施（建议 4 个迭代）

## 迭代 A（3-4 天）：数据层 + 选股器 MVP
- 交付：
  - 数据接入抽象 + 基础缓存。
  - 选股器筛选、排序、方案保存。

## 迭代 B（3-4 天）：股票分析 MVP
- 交付：
  - 单票分析页。
  - 2-3 票对比功能。

## 迭代 C（3 天）：财务仪表盘 + 价格预警 MVP
- 交付：
  - 财务趋势面板。
  - 预警规则与触发列表。

## 迭代 D（3-4 天）：AI 助手 MVP
- 交付：
  - 检索增强问答。
  - 引用回链与基础会话记录。

## 6. 依赖与风险

- 外部数据源稳定性：需做缓存与降级。
- 指标口径一致性：统一计算公式，避免模块间结果不一致。
- 本地性能：先限制股票池规模和图表刷新频率，避免卡顿。
- AI 幻觉风险：必须强制引用来源，无来源不输出结论型答案。

## 7. 完成定义（DoD）

- 五个模块均从占位页升级为可操作页面。
- 每个模块至少有一个可验收的闭环流程。
- lint/typecheck 通过，关键路径有最少测试覆盖。
- 不依赖手工改代码即可在页面完成基本操作。

- 原因：它能沉淀通用数据层，后续股票分析/仪表盘/预警都能复用。
